# 周开发进度汇报生成助手

根据用户录入的开发工作内容自动生成格式化的周开发进度汇报。

## 功能说明

- 自动解析按业务域/模块记录的开发工作内容
- 智能分类整理（按业务域分大类，按模块/重点任务分小类）
- 识别任务状态（计划中 / 进行中 / 已完成）
- 生成标准格式的开发进度汇报文本（Markdown表格形式）

## 使用方法

1. 用户粘贴本周开发工作内容（按业务域/模块记录的任务清单）
2. 自动解析并生成分类整理的进度汇报
3. 输出格式化的进度汇报文本

## 输入格式

用户输入通常是按业务域/模块记录的开发工作内容，格式如下：

```
### 本周开发进度汇报

---

#### 一、AI编程能力

| 重点任务              | 具体内容                                 | 开发进度 |
| --------------------- | ---------------------------------------- | -------- |
| Markdown流式输出      | 完成token提供，进入视觉设计              | 进行中   |
| AI Sender输入框组件   | 完成文档、skill编写及最终验收            | 进行中   |
| 思维链与思考过程      | 完成验收                                 | 已完成   |

---

#### 二、组件库

| 模块             | 重点任务               | 具体内容                                  | 开发进度             |
| ---------------- | ---------------------- | ----------------------------------------- | -------------------- |
| **表格组件**     | 基础表格               | 开发footer、表头自定义（slot）、行/列合并 | 计划中               |
|                  | 树形表                 | 支持筛选、排序、虚拟滚动并验收            | 计划中               |
| **新组件开发**   | 长日期范围选择器       | 跨端自适应改造                            | 计划中               |

---

#### 五、关注项与依赖

- **导航组件走查问题**：目前修复进度较慢，需重点跟进验收。
- **基础资料视觉设计**：本周暂缓，后续根据优先级再启动。
```

## 处理流程

### 第一步：内容解析

1. **业务域识别**：识别 `#### 一、XXX` / `#### 二、XXX` 格式的业务域标题
2. **表格解析**：提取表格中的任务数据
   - 识别表头：重点任务/模块/具体内容/开发进度
   - 识别行数据：每一行对应一个任务
3. **状态标记识别**：
   - `计划中` / `待开始` 表示计划中
   - `进行中` / `开发中` 表示进行中
   - `已完成` / `已验收` / `完成` 表示已完成
4. **关注项识别**：识别 `#### 五、关注项与依赖` 下的列表项

### 第二步：任务分类

根据业务域将任务归类：

**常见业务域**
- AI编程能力
- 组件库
- 脚手架
- 网站 / 官网
- 其他业务域...

**每个业务域下的子分类**
- 按"模块"或"重点任务"分组
- 提取"具体内容"作为任务描述
- 保留"开发进度"状态

### 第三步：内容整理

1. **保留业务域结构**：按原业务域分类展示
2. **简化描述**：保留"重点任务"+"具体内容"的核心信息
3. **状态统一**：统一进度状态描述（计划中/进行中/已完成）
4. **关注项提取**：单独提取关注项与依赖事项

### 第四步：格式化输出

按照标准进度汇报模板输出：

```
### 本周开发进度汇报

---

#### 一、[业务域名称]

| 重点任务/模块 | 具体内容 | 开发进度 |
|--------------|----------|----------|
| [任务名称] | [任务描述] | [状态] |
| [任务名称] | [任务描述] | [状态] |

---

#### 二、[业务域名称]

| 模块 | 重点任务 | 具体内容 | 开发进度 |
|------|----------|----------|----------|
| [模块名] | [任务名] | [描述] | [状态] |

---

#### 五、关注项与依赖

- **[事项名称]**：[具体说明]
- **[事项名称]**：[具体说明]
```

## 输出格式规范

### 业务域展示

每个业务域以 `#### 一、XXX` 形式展示，包含：
- 业务域标题
- 任务表格（根据内容选择3列或4列表格）

### 表格列规范

**3列表格**（简单任务）：
| 重点任务 | 具体内容 | 开发进度 |

**4列表格**（复杂任务，含模块分组）：
| 模块 | 重点任务 | 具体内容 | 开发进度 |

### 状态显示

- **计划中**：灰色或正常显示
- **进行中**：正常显示
- **已完成**：正常显示

### 关注项与依赖

以列表形式展示：
- **[事项名称]**：[具体说明]

## 状态标记说明

| 标记 | 含义 | 处理方式 |
|------|------|----------|
| 计划中 / 待开始 | 尚未开始 | 保留原样 |
| 进行中 / 开发中 | 正在进行 | 保留原样 |
| 已完成 / 已验收 / 完成 | 已完成 | 保留原样 |
| xx% | 具体进度 | 保留百分比 |

## 注意事项

1. 保留业务域分类结构（一、二、三...）
2. 表格格式保持对齐，使用标准Markdown表格语法
3. 模块列如为空，保持为空或继承上方模块名
4. 关注项与依赖单独展示在"五、关注项与依赖"部分
5. 状态描述保留原样（计划中/进行中/已完成）
6. 括号内的补充说明（如"今日完成"、"目标100%"）保留
