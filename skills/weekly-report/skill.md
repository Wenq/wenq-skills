# 周报生成助手

根据用户录入的周工作内容自动生成格式化工作周报。

## 功能说明

- 自动解析按日期记录的工作内容
- 智能分类整理（补丁与立项 / KWC / 需求对接 / 其他）
- 识别任务状态（重要🌟 / 完成 / 进行中）
- 生成标准格式的周报文本

## 使用方法

1. 用户粘贴本周工作内容（按日期记录的任务清单）
2. 自动解析并生成分类整理的周报
3. 输出格式化的周报文本

## 输入格式

用户输入通常是按日期记录的零散工作内容，格式如下：

```
2026/3/16（周一）：
1. 任务A 🌟
2. 任务B -ok
3. 任务C -?

2026/3/17：
1. 任务D
```

## 处理流程

### 第一步：内容解析

1. **日期识别**：识别 `YYYY/M/D（周X）` 格式的日期行
2. **任务提取**：提取编号列表项（1. 2. 3.）作为任务
3. **状态标记识别**：
   - 🌟 表示重要事项
   - `-ok` / `-100%` 表示已完成
   - `-?` / `-50%` 表示进行中/有阻塞
4. **子任务处理**：处理缩进的子项内容

### 第二步：任务分类

根据关键词将任务归类到以下模块：

**补丁与立项**
- 关键词：补丁、hotfix、紧急修复、立项、迭代、基线、回迁、发布说明

**KWC**
- 关键词：KWC、组件、控件、网站、业务组件、F7、markdown、晨会

**需求对接**
- 关键词：需求、验收、评审、接口、API、伙伴、客户支持

**其他**
- 未匹配以上内容统一归入此项

### 第三步：内容整理

1. **简化描述**：去除层级编号，保留核心描述
2. **合并重复**：同一任务跨多天记录合并为一条
3. **提取阻塞项**：标记为 `-?` 的任务归入"阻塞/风险"模块
4. **生成计划**：提取标记为 🌟 的重要事项作为下周计划参考

### 第四步：格式化输出

按照标准周报模板输出：

```
1、本周重点进展
补丁与立项
- 任务1
- 任务2

KWC
- 任务3

需求对接
- 任务4

其他
- 任务5

2、当前阻塞/风险/需支持事项
- 任务X（进行中）

3、下周核心计划
- 重点任务
```

## 输出格式规范

### 1、本周重点进展

#### 补丁与立项
- 任务描述（简洁明了）

#### KWC
- 任务描述

#### 需求对接
- 任务描述

#### 其他
- 任务描述

### 2、当前阻塞/风险/需支持事项
- 任务描述（状态）

### 3、下周核心计划
- 计划事项

## 示例

### 输入示例

用户粘贴：

```
2026/3/16（周一）：
1. 专利 - 技术交底书 （页面脚本）🌟
2. 紧急补丁 - A产品海外客户，text/json -》 application/json -50%？🌟
3. KWC - 网站内容补全
4. 技术支持 - 在线预览不能打印

2026/3/17：
1. KWC - 开发进度沟通、网站图标大小改为24px -ok
2. 补丁 - 基线验证kwc内容 - ？
```

### 输出示例

```
1、本周重点进展
补丁与立项
- A产品海外客户紧急补丁支持（text/json 修改为 application/json）
- 基线验证kwc内容

KWC
- 网站内容补全
- 开发进度沟通、网站图标大小改为24px

其他
- 专利 - 技术交底书（页面脚本）
- 技术支持 - 在线预览问题处理

2、当前阻塞/风险/需支持事项
- 基线验证kwc内容（进行中）

3、下周核心计划
- 专利 - 技术交底书完善
```

## 状态标记说明

| 标记 | 含义 | 处理方式 |
|------|------|----------|
| 🌟 | 重要事项 | 保留标记或优先展示 |
| -ok / -100% | 已完成 | 正常展示，可标注完成状态 |
| -? / -50% | 进行中 | 归入"阻塞/风险"事项 |
| （人名） | 协作人 | 保留信息 |

## 注意事项

1. 忽略具体日期，按任务类别聚合
2. 去除层级编号和多余缩进
3. 简化长描述，保留核心信息
4. 提取【问题描述】【修复方案】等关键信息
5. 同一任务跨多天记录的，合并为一条并保留最新状态
