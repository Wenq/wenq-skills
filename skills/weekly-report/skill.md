# 周报生成助手

## 元信息

- **名称**: weekly-report
- **版本**: 1.0.0
- **描述**: 根据用户录入的周工作内容自动生成格式化周报
- **作者**: Alan
- **支持IDE**: Qoder, Trae, Cursor, VSCode

## 触发方式

用户输入以下关键词触发：
- `/周报`
- `/weekly`
- `生成周报`
- `整理周报`

## 使用方式

1. 用户粘贴本周工作内容（按日期记录的任务清单）
2. Skill 自动解析并生成分类整理的周报
3. 输出格式化的周报文本

## 处理流程

### 第一步：识别输入内容

用户输入通常是按日期记录的零散工作内容，格式如下：

```
2026/3/16（周一）：
1. 任务A 🌟
2. 任务B -ok
3. 任务C -?

2026/3/17：
1. 任务D
```

### 第二步：内容解析规则

1. **日期识别**：识别 `YYYY/M/D（周X）` 格式的日期行
2. **任务提取**：提取编号列表项（1. 2. 3.）作为任务
3. **状态标记识别**：
   - 🌟 表示重要事项
   - `-ok` / `-100%` 表示已完成
   - `-?` / `-50%` 表示进行中/有阻塞
4. **子任务处理**：处理缩进的子项内容

### 第三步：任务分类

根据关键词将任务归类到以下模块：

**补丁与立项**
- 关键词：补丁、hotfix、紧急修复、立项、迭代、基线、回迁、发布说明

**KWC**
- 关键词：KWC、组件、控件、网站、业务组件、F7、markdown、晨会

**需求对接**
- 关键词：需求、验收、评审、接口、API、伙伴、客户支持

**其他**
- 未匹配以上内容统一归入此项

### 第四步：内容整理

1. **简化描述**：去除层级编号，保留核心描述
2. **合并重复**：同一任务跨多天记录合并为一条
3. **提取阻塞项**：标记为 `-?` 的任务归入"阻塞/风险"模块
4. **生成计划**：提取标记为 🌟 的重要事项作为下周计划参考

### 第五步：格式化输出

按照标准周报模板输出：

```
1、本周重点进展
补丁与立项
- 任务1
- 任务2

KWC
- 任务3

需求对接
- 任务4

其他
- 任务5

2、当前阻塞/风险/需支持事项
- 任务X（进行中）

3、下周核心计划
- 重点任务
```

## 示例

### 输入示例

用户粘贴：

```
2026/3/16（周一）：
1. 专利 - 技术交底书 （页面脚本）🌟
2. 紧急补丁 - A产品海外客户，text/json -》 application/json -50%？🌟
3. KWC - 网站内容补全
4. 技术支持 - 在线预览不能打印

2026/3/17：
1. KWC - 开发进度沟通、网站图标大小改为24px -ok
2. 补丁 - 基线验证kwc内容 - ？
```

### 输出示例

```
1、本周重点进展
补丁与立项
- A产品海外客户紧急补丁支持（text/json 修改为 application/json）
- 基线验证kwc内容

KWC
- 网站内容补全
- 开发进度沟通、网站图标大小改为24px

其他
- 专利 - 技术交底书（页面脚本）
- 技术支持 - 在线预览问题处理

2、当前阻塞/风险/需支持事项
- 基线验证kwc内容（进行中）

3、下周核心计划
- 专利 - 技术交底书完善
```

## 提示词模板

当用户触发 Skill 时，使用以下提示词指导 AI 处理：

```
你是一位专业的周报整理助手。请根据用户提供的周工作内容，按照以下规则生成格式化周报：

## 处理步骤

1. **解析输入内容**
   - 识别日期行（格式：YYYY/M/D（周X））
   - 提取所有任务项（编号列表）
   - 识别状态标记：🌟（重要）、-ok（完成）、-?（进行中）

2. **任务分类**
   将任务按以下类别归类：
   - **补丁与立项**：包含"补丁"、"hotfix"、"紧急"、"迭代"、"基线"、"回迁"、"发布"等关键词
   - **KWC**：包含"KWC"、"组件"、"网站"、"F7"、"markdown"、"晨会"等关键词
   - **需求对接**：包含"需求"、"验收"、"接口"、"伙伴"等关键词
   - **其他**：未匹配以上内容

3. **内容整理**
   - 去除层级编号和多余缩进
   - 合并重复任务，保留最新状态
   - 简化长描述，保留核心信息
   - 提取【问题描述】【修复方案】等关键信息

4. **识别阻塞事项**
   - 提取标记为"-?"或进行中的任务
   - 归入"当前阻塞/风险/需支持事项"

5. **生成下周计划**
   - 提取标记为🌟的重要事项
   - 延续本周未完成的重点工作

## 输出格式

```
1、本周重点进展
[分类名]
- [任务描述]

2、当前阻塞/风险/需支持事项
- [任务描述]（[状态]）

3、下周核心计划
- [计划事项]
```

请直接输出生成的周报内容，不需要解释处理过程。
```
