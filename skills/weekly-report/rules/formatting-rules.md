# 周报格式化规则

## 一、内容分类规则

### 1. 补丁与立项
**关键词匹配：**
- 补丁、hotfix、紧急修复
- 立项、需求梳理、迭代
- 基线、回迁、发布说明
- 版本、发布

**归类内容：**
- 各类补丁开发、验证、发布
- 迭代需求梳理和跟进
- 基线验证和回迁
- 发布相关文档编写

### 2. KWC
**关键词匹配：**
- KWC、组件、控件
- 网站、官网、文档
- 业务组件、F7、markdown
- 晨会、开发进度

**归类内容：**
- KWC项目相关开发工作
- 组件开发和维护
- 网站内容维护
- 需求沟通和进度同步

### 3. 需求对接
**关键词匹配：**
- 需求、验收、评审
- 接口、API、对接
- 伙伴、客户支持

**归类内容：**
- 需求评审和验收
- 接口对接支持
- 伙伴技术支持

### 4. 其他
**未匹配到以上分类的内容统一归入此项**
- AI工具体验
- 专利、技术交底书
- 技术支持、问题处理
- 会议、培训等

## 二、状态标记识别

| 标记 | 含义 | 处理方式 |
|------|------|----------|
| 🌟 | 重要事项 | 保留标记或优先展示 |
| -ok / -100% | 已完成 | 正常展示，可标注完成状态 |
| -? / -50% | 进行中 | 归入"阻塞/风险"事项 |
| （人名） | 协作人 | 保留信息 |

## 三、内容整理规则

### 1. 日期信息处理
- 忽略具体日期，按任务类别聚合
- 保留时间顺序（先记录的内容排在前面）

### 2. 任务描述简化
- 去除层级编号（1. 2. 3.）
- 去除空行和多余缩进
- 保留核心描述信息
- 去除重复前缀（如多个"补丁"相关合并）

### 3. 详细说明处理
- 【问题描述】【原因说明】【修复方案】等详细内容
- 提取核心要点，简化为一行描述
- 技术细节可适当保留

### 4. 重复项合并
- 同一任务跨多天记录的，合并为一条
- 保留最新进展状态

## 四、输出格式规范

### 1. 模块标题
使用三级标题（###）标识分类

### 2. 列表格式
- 使用 `- ` 作为列表前缀
- 每条任务独立一行
- 相关任务可合并描述

### 3. 阻塞事项格式
- 提取标记为"-?"或进行中的任务
- 格式：`任务描述（状态）`

### 4. 下周计划
- 提取标记为🌟的重要事项
- 延续本周未完成的重点工作
