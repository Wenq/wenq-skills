# 周报格式化规则

## 一、内容分类规则

**关键词匹配：**
- KWC、组件、控件
- 网站、官网、文档

- 晨会、开发进度

**归类内容：**
- KWC项目相关开发工作
- 组件开发和维护
- 网站内容维护
- 需求沟通和进度同步

### 2. 低代码相关

- 伙伴支持、客户沟通
- 需求对齐、确认

**归类内容：**
- 需求评审和验收
- 接口对接支持
- 伙伴/客户技术支持
- 需求沟通与对齐

#### 2.2 需求规划
**关键词匹配：**
- 需求梳理、迭代规划
- 方案设计、需求分析
- 需求完善、规划

**归类内容：**
- 需求梳理和分析
- 方案设计与规划
- 迭代需求规划

### 3. 项目组事务

## 二、状态标记识别

| 标记 | 含义 | 处理方式 |
|------|------|----------|
| 🌟 | 重要事项 | 保留标记或优先展示 |
| -ok / -100% | 已完成 | 正常展示，可标注完成状态 |
| -? / -50% | 进行中 | 归入"阻塞/风险"事项 |
| （人名） | 协作人 | 保留信息 |

## 三、内容整理规则

### 1. 日期信息处理
- 忽略具体日期，按任务类别聚合
- 保留时间顺序（先记录的内容排在前面）

### 2. 任务描述简化
- 去除层级编号（1. 2. 3.）
- 去除空行和多余缩进
- 精简为一句话核心描述，去掉技术细节和过程信息
- 去除重复前缀（如多个"补丁"相关合并）
- 去除【问题描述】【原因说明】【修复方案】等详细说明，仅保留结论

### 3. 重复项合并
- 同一任务跨多天记录的，合并为一条
- 保留最新进展状态

## 四、子类别自动生成规则

1. **触发条件**：某大类下有 2 条及以上任务，且任务间存在明显主题差异
2. **命名规范**：用简短主题名（2-6个字），如"网站维护""接口排查""插件问题"
3. **优先复用预设**：各大类的预设子类别优先使用，无法归入时才新建
4. **不强制生成**：任务少（仅1条）或主题单一时，直接列在大类下
5. **全局适用**：所有 5 个大类均可自动生成子类别

## 五、输出格式规范

### 1. 模块标题
- 一级分类（大类）独占一行
- 二级分类（子类别）用两个空格缩进，加粗显示

### 2. 列表格式
- 使用 `- ` 作为列表前缀
- 每条任务独立一行
- 二级分类下的列表项前增加两个空格缩进
- 相关任务可合并描述

### 3. 阻塞事项格式
- 提取标记为"-?"或进行中的任务
- 格式：`任务描述（状态）`

### 4. 下周计划
- 提取标记为🌟的重要事项
- 延续本周未完成的重点工作
