# 一、功能概述
通过前端页面脚本扩展功能，用户可以直接编写javascript代码，从而快速实现特定的业务控制逻辑
易于使用，对现有业务逻辑进行扩展操作简单（前提掌握脚本开发）
部署方便，脚本跟随元数据一起导入导出、补丁升级

支持内置API和组件事件等，增强了前端页面定制化开发能力
**如想快速查看相关定制效果，可直接查看章节九"**[案例展示](https://vip.kingdee.com/article/588370901062560768?productLineId=29&amp;isKnowledge=2&amp;lang=zh-CN#61)**"阅读对应内容**

# 二、适用场景
以下场景可以考虑使用页面脚本扩展功能来实现：
1、想在前端页面快速实现一些简单的业务控制逻辑
2、依据页面数据状态等需要对某些UI元素做定制开发（比如修改内容展示格式，样式以及动画等）
3、开发的自定义控件其渲染数据依赖页面上字段等数据的变化，不想通过服务端插件实现
4、其他基于web页面轻量化JS脚本想实现的功能等
**注意**：
1、标准业务与二开项目等都可使用（有做权限控制）
2、**脚本在设计器中可进行编辑或查看，且考虑到web页面脚本的安全性等，不建议将重要或复杂的业务逻辑放在脚本中实现**
3、**如果想开发一个独立控件且需要与服务端插件通信，同时希望在不同单据页面可复用，则推荐使用[自定义控件](https://developer.kingdee.com/knowledge/specialDetail/218022218066869248?category=249447947539332864&amp;id=205421332635100416&amp;productLineId=29)做开发**

# 四、脚本编写规范与语法支持
脚本编写使用javascript，内置了一些脚本对象、API和标准事件等。
**一个单据页面自身只会有一份JS脚本，同时支持通过单据继承/扩展/复制等方式来继承父页面的脚本**。
## 脚本生命周期
![](http://fd.kingdee.com:9001/file/image/66b46936e764ea00c3a2616b)
脚本编写主要在两个大的生命周期函数中进行，分别为：didMount、willUnmount
### didMount

    * 单据初始化函数
    * 执行初始化相关事务，比如注册组件事件监听、设置数据、执行DOM操作等
    * 此函数在单据加载服务端插件后自动调用

**注意**：
**1、一般脚本代码的主要实现逻辑都建议在该函数中编写实现**
**2、该函数在页面业务数据加载完成之后执行（即"loaddata"请求）**
### willUnmount

    * 单据卸载函数
    * 执行资源释放等事务，比如移除DOM事件监听/定时器、销毁UI元素、删除全局变量等
    * 此函数在单据关闭(销毁)时自动调用

**注意：该函数在页面关闭时执行**
## 事件与功能接口
支持范围：包括单据页面和各类控件，PC和移动端
事件：用于在页面脚本中做监听，特定操作或数据状态下会触发，在其回调方法中实现自己的业务逻辑
功能接口：控件对象对外暴露的功能接口，通过在页面脚本中调用该接口可使得对应控件实现特定的业务逻辑

### 自定义控件
#### onCustomMsgEvent
自定义控件的自定义消息订阅事件，设计器中选中自定义控件，右侧"**动作**"面板可看到对应事件绑定入口

    * 在自定义控件内部调用"**this.model.triggerCustomMsgEvent(xx, xx)"接口触发该事件（即实现自定义控件内部向页面脚本中发送消息，接口调用时机和参数由开发者自定义）**
    * 基于该事件可以按需从自定义控件内部向页面脚本中发送消息，且消息内容可自定义
    * 支持在卡片分录中使用（如果在卡片分录中使用，则事件参数多一个rowIndex参数，代表所在分录行）

示例：
【页面脚本 - 事件绑定代码】
```
//页面脚本中事件绑定代码。参数结构说明：data: {type, args}，type和args参数都不一定存在，看自定义控件内部传递的参数
this.$('自定义控件标识').onCustomMsgEvent((data) => {

})

```
【自定义控件内部 - 发送消息/触发事件代码】
```
//调用示例
this.model.triggerCustomMsgEvent('dataChange', {name: '名称1', code: '888'}); //调用：接口方式一
this.model.triggerCustomMsgEvent({type: 'dataChange', name: '名称1', code: '888'}); //调用：接口方式二

```

#####setCellRender
功能接口，用于表格单元格（**单元格查看时，即非编辑态**）自定义渲染或扩展

    * 支持重新渲染整个单元格
    * 支持基于现有单元格渲染扩展新的UI元素

**接口说明**

// 接口参数，自定义渲染器即一个render函数（该函数的参数中包含单元格等相关信息）或{}对象
CustomCellRender
// 使用方式1: 自定义渲染所有单元格

// 使用方式2：按特定列自定义渲染单元格

    列id1: Render渲染函数1,
    列id2: Render渲染函数2
})

```
**具体案例见章节九 “案例展示 - 表格单元格自定义渲染或扩展”部分内容 **
**注意：**
1、Render函数“CustomCellRender”的参数props中有新增接口“isCellLock”，用于获取单元格是否锁定，根据该状态值渲染查看态UI效果
2、单元格锁定状态下，是无法进入编辑态的，所以编辑器的自定义渲染不用处理锁定状态渲染
3、多脚本（如继承单据）渲染器生效规则：同一个表格控件的同一列，子页面脚本**覆盖**父页面脚本的自定义渲染器
4、【**重要！**】使用该接口基于单元格渲染器Render扩展单元格时，从props中获取的“**value**”属性为**格式化后的值（用于在单元格中显示，非原始值；原始值要从props中的属性“originValue”获取）**，该值内容格式随“国际化”设置而变化，比如数字类型字段其值为"0.7"，设置国际化之后可能变为",7"，所以对value属性值进行处理时请谨慎，以防出现程序异常等情况

## 内置对象
"**this**"：页面脚本上下文对象，用于访问页面内所有控件实例和获取页面自身相关数据。

    * 该对象一般在didMount或willUnmount中使用，使用时注意要保证其指向正确（特别是在嵌套函数中），否则脚本会报错，无法正确访问相关对象或接口

示例：
```
 function didMount() {
     // this指向页面脚本上下文
     this.xxx
     
     //错误写法
     this.$('控件标识').on('click',function(e){
         //此时this不指向页面脚本上下文，js会报错
         this.$('控件标识').set('Name','title')
     })
     
     //正确写法（使用箭头函数，自动绑定上下文）
     this.$('控件标识').on('click',(e)=>{
         //此时this指向页面脚本上下文,js正常执行
         this.$('控件标识').set('Name','title')
     })
 }
 
 function willUnmount() {
     // this指向页面脚本上下文
     this.xxx
 }
 
 // 此时this不指向页面脚本上下文（指向window对象）
 this.xxx

```
"**this.$('控件标识')**"：用于获取控件实例，使用该实例可调用控件相关功能接口（包括通用控件方法、字段方法等）
示例：
```
// 获取控件实例对象
this.$('控件标识')

//调用控件实例对象功能接口
this.$('控件标识').set('Name', 'title')


```

## "export"关键字
如果将某些逻辑封装为独立的函数，且函数中用到了脚本内置对象或API等，为了在函数中能正常访问这些对象或API等，则需要为函数增加**export**修饰关键字，同时函数调用方式为:  **this.独立函数名称()**。
示例:
```
  // 初始化
 function didMount() {
     this.testMethod() //调用函数
 }

// 独立函数写法
export function testMethod(str) {
  this.$('控件标识').setValue('111')
}

### on
用于为DOM元素添加事件监听，接受三个参数：事件名称（HTML原生事件，可同时监听多个，用空格分隔）、子元素选择器 【可选】、事件回调函数

示例：
```

```
**注意：**如果需要监听控件中某个子元素的事件，就需要用到子元素选择器selector，但其中获取焦点和失去焦点事件"focus、blur"的写法特殊（事件名称要替换一下"focus -&gt; focusin、blur -&gt; focusout"），具体参考如下：
```
//写法推荐
//获取焦点
const handler=event=>{...}
this.$('textfield1').on('focusin', 'input', handler)
//失去焦点
this.$('textfield1').on('focusout', 'input', handler)

//不生效写法
this.$('textfield1').on('focus', 'input', handler)
this.$('textfield1').on('blur', 'input', handler)

```

【功能场景】：需要将用户录入的阿拉伯数字自动转为中文数字

/**
 * 将字符串中数字转换为大写数字
 */
function convertToChineseNumber(str) {
  // 正则表达式匹配字符串中的所有数字
  const regex = /&#92;d+/g;
  // 替换函数，将匹配到的数字转换为中文数字
  return str.replace(regex, (match) => {
    const num = parseInt(match, 10); // 将匹配到的字符串转换为数字
    const chineseNumbers = ['零', '壹', '贰', '叁', '肆', '伍', '陆', '柒', '捌', '玖'];
    // 将数字转换为中文数字字符串
    let chineseStr = '';
    for (let i = 0; i < num.toString().length; i++) {
      chineseStr += chineseNumbers[parseInt(num.toString()[i], 10)];
    }
    return chineseStr;
  });
}

# 十一、注意事项
1、IE浏览器暂不支持脚本编辑，可执行脚本。
2、'**set**'接口设置属性可能会被服务端插件设置内容覆盖（原因是服务端插件修改生效优先级&gt;=页面脚本）。
3、脚本执行异常（未捕获的），则后续脚本执行中断。
4、通过'**on**'接口监听组件DOM事件，该事件不会与组件内部状态联动。比如锁定的按钮，通过on接口监听其'click'点击事件，即使按钮通过被锁定了，事件依然会触发。
5、**'set'接口支持的参数（控件属性）有限，一般都是与样式设置相关**。参数获取示例：修改控件宽度，其参数获取方法（设计器中鼠标放到对应属性名称上即可查看到参数名称“Width”），如下截图：
![](http://fd.kingdee.com:9001/file/image/66b46a78e764ea00c3a26186)
**备注：为保证已有业务逻辑的正确性，锁定性、可见性、必录、控件标识、类型等控件属性不支持通过脚本修改**
6、'**createStyle**'接口创建Class限制其作用域在当前单据页面(防止全局样式污染)。
7、使用'**setValue**'接口为基础资料类型字段赋值会引发服务端报错，原因是基础资料字段赋值是要在服务端先赋值，再刷新前端组件（这与文本字段等不同）。
8、**历史单据新增页面脚本，脚本保存之后需要再做一次设计器的保存，否则脚本内容会丢失**。
9、脚本中标准事件或其他API接口（比如on）中使用的回调函数建议使用箭头函数写法，可自动绑定上下文，保证在函数中可正常访问脚本内置对象等。

# 十二、常见问题与解决方法
## 1、怎样获取控件Dom对象？
【解决办法】：使用页面脚本中提供的标准接口getElement或htlm的一些原生接口都可以实现
【代码示例】：
```
//【推荐】通过页面脚本标准接口（获取整个控件的dom对象）
const domEle = this.$('控件标识').getElement() //同步
this.$('控件标识').wait().then(ctrlDomEle => { }) //异步
或者 
//通过html原生接口（根据页面上页面上控件对应Dom结构里的'id'属性获取，一般就是控件标识）
const element = document.getElementById('元素ID');
或者
//其他可参考的原生接口如下
const elements = document.getElementsByClassName('类名'); //根据class类名
const elements = document.getElementsByTagName('标签名'); //根据元素类型
const element = document.querySelector('CSS选择器'); //获取第一个
const elements = document.querySelectorAll('CSS选择器'); //获取多个


```

## 2、在didMount里获取控件Dom对象是空的？
【原因分析】：
1、控件布局在容器中，但容器开启了懒加载，导致页面初始化时容器中控件可能没有立即渲染，就没有对应Dom信息。
2、控件较复杂，初始化耗时长一些导致其挂在到页面Dom结构上慢一些，此时获取不到对应Dom对象
【解决办法】：
1、**【推荐】**在单据对应表单设计器中，使用'wait'接口异步获取控件Dom对象。
示例代码：
```
//ctrlDomEle即为控件对应的Dom对象
this.$('控件标识').wait().then(ctrlDomEle => { })

```
2、复杂控件如表格和树等，都有对外提供一个“控件初始化”事件，可在该事件中获取其Dom对象
3、如果是其他场景无法获取到控件对应Dom对象，可以尝试在页面脚本didMount中使用setTimeout延迟获取，参考示例代码如下
```
//在页面初始化时延迟300ms获取dom对象
setTimeout(() => {
  cont domEle = this.$('控件标识').getElement()
  }, 300);


```

## 3、怎样监听原生html事件？
【解决办法】：使用页面脚本中提供的标准接口'on'或htlm的一些原生接口都可以实现
【代码示例】：
```
//【推荐】通过页面脚本标准接口（直接监听整个控件的某些事件，或者控件内部某个子元素的事件）
this.$('控件标识').on('click', (e)=>{ })
或者
//通过html原生接口（先获取要监听目标对应dom对象，监听对应事件）
cont domEle = this.$('控件标识').getElement()
domEle.addEventListener('click', function(event) { })


```

## 4、怎样通过页面脚本修改页面内特定控件的样式？
【解决办法】：使用页面脚本标准接口'css'或'createStyle'
【代码示例】：
```
//css接口（内容格式跟原生css一致），修改标识为'kdtest_beizhu'的控件样式
this.$('kdtest_beizhu').css({ 'color': 'yellow' })
或者
//createStyle接口（该接口作用范围为当前单据页面），修改标识为'kdtest_beizhu'的字段上标题的样式
this.utils.createStyle('[id=kdtest_beizhu] .kd-cq-field-title-wrap{font-Size:30px !important}')


```

## 5、怎样排查页面js脚本在单据页面上不生效的问题？
【原因分析】
问题1、页面没有加载JS脚本或内容加载异常
问题2、页面加载的JS脚本内容不是最新的
问题3、编写的JS脚本内容与浏览器或具体业务数据相关，运行时获取到相关对象不存在导致程序执行失败
【解决办法】
问题1、通过脚本内容接口请求，确认是否加载以及加载脚本内容是否异常
**通过浏览器控制台查看加载JS脚本请求接口"getPageJs.do"（带有页面标识等信息）**：
前端页面会缓存已加载脚本数据，如果脚本内容无变化，下次打开单据不会再发送该接口请求（后续脚本内容有变化或清理掉浏览器缓存，再打开单据就会发送该接口请求）。
